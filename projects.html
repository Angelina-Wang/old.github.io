Scheme Interpreter

Building a Scheme interpreter in Python was a challenge because it was my first major cs project and I had to fully understand how languages are implemented--and then subsequently implement that myself. The toughest part was picturing how each part that I was implementing would fit into the interpreter as a whole, so that each "piece" of the input that I separated would be redirected to the proper function. One topic I struggled with initially was understanding recursion. It seemed so strange that a method could call, and consequently use, itself. I'm pretty sure I had a (not so) similar struggle in second grade when I played soccer and tried to do a throw-in to myself. Except in computer science, unlike soccer, using yourself is allowed. Anyway, some drawn out environment diagrams and accidental infinite loops later, it all made sense.

Text Editor

A text editor built from scratch that has clicking, cursor, scroll bar, word wrap, open/save, size, and undo/redo capabilities. I will never forget the moment I got the cursor to blink consistently in a way that did not make me dizzy.

Bear Maps

A version of Google Maps (or Apple Maps, depending on your loyalties) for the Berkeley area that used the A* search algorithm to locate the most efficient routes from one destination to another. I rastered using a quad tree made up of images with varying amounts of detail, depending on the amount the user zoomed in. When searching for locations in the area using the search bar, a trie of all the locations provides a list of suggestions that match the prefix inputted.
